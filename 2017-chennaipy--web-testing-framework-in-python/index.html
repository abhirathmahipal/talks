Web Testing Framework in Python

by Abhirath Mahipal

Need

  1. Repetitive tasks like clicking, inputting and checking simple conditions.
  2. RELATIVELY easier path for non programmers to write test cases in a language which is close to English
  3. Easily adding more functionality for use by more experiencesd users.
  4. Headless testing, parallel test execution and easy to digest reports.

Structure

  • Tests are written in Lisp
  • Pre-processing stage.
    1. Imports - simple read and append to list.
    2. All functions are stripped into list entries.
    3. Irrelevant stuff removed. The end solution isn't very elegant. Just pieces forced to work due to lack of insight and options.

continued...

  • hy generates JSON in accordance to function declarations.
  • JSON is translated into appropriate selenium commands.
  • Test analysis reports are rendered using Jinja templates.

Dictionaries vs Lists

  • For our given purposes consider Dictionaries to represent Abstract Syntax Trees.
  • Dictionaries are most likely to be used in YAML or JSON to write the test cases. This forces a person to remember and write both the key and the value. Lists could be used but they become difficult to validate (has the user entered everything that needs to be entered etc).
  • Validation functions for dictionaries are easier. Keys are likely to be repeated in every API. Common functions can be written.

continued...

  • The number of arguments taken by click differs from that of input. A new function is required to validate each and every API provided by Glitch
  • Say we use a particular value often and want to set a default. We could use a simple check to see if the key exists. If it doesn't provide a default value.
  • Lists force a person to remember the order of the arguments.

Why Lisp?

  • Well suited for custom syntax and query languages. FYI:- Lisp is a versatile choice when it comes to writing query languages similar to those used to query MySQL, sqlite etc.
  • Very limited language grammar.
  • Easy to read.
  • Libraries to parse Lisp in Python exist.
  • The ease of using existing code surpasses the little benefit over a custom built parser. No regex, CFG and mind bloggling loops.
